linux syscalls :

        socket() -> returns a fd , fd is an index to per process file desciptor table 
                    maintained by the kernel 

                    -> It can represent various I/O resources i.e. regular files , directories, 
                        pipes  hardware devices  and network sockets.

                    -> 0 : standard input , 1: standard output and 2 standard error

        bind()   -> associates an address to a socket fd

        listen() -> enables us to accept connection to that address

        accept() -> takes a listenening fd and returns a connection fd or socket 

        read() -> returns whatever data is available in the kernel or blocks if there is none

        write() -> sends data

        close() -> destroys the resource referred by the fd and recycles the fd number

        connect() -> client side sycall that takes a socket  fd  and makes a tcp connection to that address


        AF_INET4 -> for ipv4

        AF_INET6 -> for ip6

        SOCK_STREAM -> for TCP 

        setsockopt(fd, SOL_SOCKET , SO_REUSEADDR, &val, sizeof(val));

        setsockopt -> syscall is used to configure various aspects of a socket 


BUFFERED I/0 : 

            A data tranfer technique that uses an intermediate memory area (buffer) 
            to store data temporarily allowing multiple small I/O requests to be batched 
            into fewer larger syscalls


concurrent connections in a server side network program:

    -> forking : creates new processes for each new client connection

    -> multithreading : uses thread instead of processes

    -> event loops : uses poling and non blocking I/O
        
        what is blocking mode ?

            read blocks the caller when there is no data in the kernel 

            write blocks when the buffer is full 

            accept blocks when there are no new connections in the kernel queue

        what is non blocking mode ?

                in non blocking mode these operations either succeed without blocking or fail
                with the errno EAGAIN which means not ready  
                Non blocking operations that fail with EAGAIN must be retried as readiness was not notified by the poll

    poll is the sole blocking ooperation in an event loop , everything else must be non blocking


    APIs that don't have a non blocking mode , such as gethostbyname and disk I/O should be 
    performed by thread pools 

    fcntl() : sets fd to non blocking mode

    select vs epoll : both are essentially same except select is has limited no of max fd capacity

                    epoll API is stateful

                    instead of supplying a set of fds on a sycall argument , epoll_ctl was used to manipulate a fd set

                    created by epoll_create , which epoll_wait is operating on

    epoll() : has 3 syscalls 

        -> epoll_create

        ->epoll_wait 

        ->epoll_ctl
